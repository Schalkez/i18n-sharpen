<objective>
Add a `--clean-empty` CLI flag (and `prune.cleanEmpty?: boolean` config field) to `prune`. When set AND the layout is `namespaced`, namespace files left with zero keys after pruning are physically deleted (or "Would delete" logged in dry-run). Flat layout NEVER deletes `<lang>.json` files (D-09) — users may intentionally keep an empty JSON file for git tracking. Never deletes the parent `<lang>/` directory.

This is purely opt-in additive behavior — existing prune runs without the flag behave exactly as in v0.2.x.

Purpose: NSWRITE-04 — namespace files preserved on disk unless user opts into deletion via `--clean-empty`. Prevents surprise data loss when a namespace is fully pruned.
Output: New `cleanEmpty` config + CLI flag + `cleanEmptyNamespaceFiles` helper invoked post-write in namespaced prune.
</objective>

    ```typescript
    /**
     * D-09: Delete (or in dry-run, log) namespace files that ended up with
     * zero keys after pruning. Operates on the SAME write plans that
     * `executePrunePlans` processed.
     *
     * Flat layout is exempt: callers MUST only invoke this for
     * `localesLayout: "namespaced"`. Never deletes the parent `<lang>/`
     * directory.
     *
     * @param emptyPlans  Subset of namespaced WritePlans whose `nestedJson`
     *                    has zero own-keys after prune.
     * @param dryRun      When true, log "Would delete" but do not touch disk.
     */
    export function cleanEmptyNamespaceFiles(
      emptyPlans: { langPath: string; displayName?: string }[],
      dryRun: boolean
    ): void {
      if (emptyPlans.length === 0) return

      const labels = emptyPlans.map(
        p => p.displayName ?? path.basename(p.langPath)
      )

      if (dryRun) {
        log.info(
          `Would delete ${pc.yellow(emptyPlans.length)} empty namespace file(s): ${labels.map(l => pc.cyan(l)).join(", ")}`
        )
        return
      }

      let deletedCount = 0
      for (const plan of emptyPlans) {
        try {
          // Use the imported `fs` (add `import * as fs from "fs"` at top if not already present)
          fs.unlinkSync(plan.langPath)
          deletedCount++
        } catch (error) {
          // Non-fatal: log and continue. The prune writes already succeeded;
          // we don't want a cleanup failure to mask the successful prune.
          log.warn(
            `Failed to delete empty namespace file ${pc.cyan(plan.langPath)}: ${(error as Error).message}`
          )
        }
      }
      if (deletedCount > 0) {
        log.success(
          `Deleted ${pc.yellow(deletedCount)} empty namespace file(s): ${labels.slice(0, deletedCount).map(l => pc.cyan(l)).join(", ")}`
        )
      }
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI flag → filesystem deletion | `--clean-empty` controls `fs.unlinkSync` calls; deletion targets derived from `loadNamespacedLocales` enumeration of `localesDir` (controlled config path) |
| Config file → cleanEmpty | `prune.cleanEmpty` parsed by zod boolean; no path/value injection surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T (Tampering) / D (Denial of service) | `fs.unlinkSync` deleting user files when prune mis-detects "empty" | mitigate | Helper only operates on plans whose `nestedJson` is empty AFTER prune logic ran — guaranteed via `Object.keys(p.nestedJson).length === 0` check at call site. Helper is gated behind both (a) explicit `cleanEmpty === true` flag, (b) `localesLayout === "namespaced"`, (c) executePrunePlans success (called AFTER the write phase returns). On individual unlink failure, log warning and continue (do NOT abort — the prune writes already succeeded). |
| T-03-02 | T (Tampering) | Deletion path includes `..` traversal via filename | accept | Plan filenames originate from `loadNamespacedLocales` enumeration of `localesDir` (real on-disk entries), not from user-runtime input. `path.basename`-style traversal is not possible because the `langPath` came from `path.join(localesDirAbs, lang, nsFile)` and Node's `fs.readdirSync` cannot return entries containing `/` or `\`. |
| T-03-03 | D (Denial of service) | Parent `<lang>/` directory deletion | mitigate | Helper explicitly only deletes individual files via `fs.unlinkSync`; never calls `fs.rmdirSync` or `fs.rmSync({recursive})`. Acceptance criterion: `grep -n "rmdirSync\\|rmSync" src/commands/prune/plans.ts` returns 0 matches. |
| T-03-04 | T (Tampering) | Existing prototype-pollution guards in transform.ts | accept (regression check) | Plan does NOT modify `transform.ts`. Post-edit check: `grep -n "FORBIDDEN_KEY_SEGMENTS" src/core/locale-io/transform.ts` returns ≥1 match. |
