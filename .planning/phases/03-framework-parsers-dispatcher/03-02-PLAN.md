<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.vue` source → @vue/compiler-sfc | Untrusted-shaped source string parsed by the SFC compiler |
| embedded `<script>` content → parseTypeScriptFile | Delegated parse of script block text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-V1 | Denial of Service | malformed `.vue` aborts the run | mitigate | SFC `parse()` errors are mapped into `collectedErrors` (D-17); the function never throws for a file-level syntax error, so one bad file cannot abort a scan. |
| T-03-V2 | Elevation of Privilege | parser executing parsed source | mitigate | `parseVueFile` only walks the AST and delegates to `parseTypeScriptFile` (parser-only `ts.createSourceFile`). No `eval`, no `compileTemplate`, no render-fn generation — source is never executed (grep asserts no `eval`). |
| T-03-V3 | Tampering | dynamic load of arbitrary module | mitigate | Compiler loaded only via `loadWorkspaceDep('@vue/compiler-sfc', cwd)` — a fixed package name resolved from the workspace, never a path derived from file content. |
| T-03-V4 | Denial of Service | ReDoS / quadratic blowup | accept | No regex over template content (AST walk only); `source.indexOf` is linear. Pathological-size inputs are out of scope (RESEARCH §Security Domain: WASM/large-file exhaustion deferred). |
</threat_model>
