<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.astro` source → @astrojs/compiler (WASM) | Untrusted-shaped source string parsed by a sandboxed WASM parser (parse-only; no transform/codegen, no execution) |
| embedded frontmatter (`---` TS) → parseTypeScriptFile | Delegated parse of the frontmatter block value |
| WASM init → concurrent callers | The first parse triggers WASM startup shared across concurrent calls via the module-level initPromise |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-A1 | Denial of Service | malformed `.astro` aborts the run | mitigate | `await astroCompiler.parse(...)` is wrapped in try/catch; the error is mapped into `collectedErrors` (D-17) and the function returns normally — one bad file cannot abort a scan. |
| T-03-A2 | Elevation of Privilege | parser executing parsed source | mitigate | `parseAstroFile` only walks the AST (parse-only) and delegates frontmatter to `parseTypeScriptFile` (parser-only `ts.createSourceFile`). The WASM compiler is a sandboxed parser — no `transform`/codegen, no `eval`; source is never executed (grep asserts no `eval`/no `transform(`). |
| T-03-A3 | Tampering | dynamic load of arbitrary module | mitigate | Compiler loaded only via `loadWorkspaceDep('@astrojs/compiler', cwd)` — a fixed package name resolved from the workspace, never a path derived from file content. |
| T-03-A4 | Denial of Service | concurrent WASM init race / corrupt shared state | mitigate | Module-level `initPromise` singleton serializes the first WASM startup; all concurrent callers await one settled promise (Pitfall 6). The 10-concurrent test asserts identical results, catching any race regression. |
| T-03-A5 | Denial of Service | WASM memory exhaustion on very large input | accept | Out of scope per RESEARCH §Security Domain (Astro compiler internal; no Phase 3 mitigation). AST walk + `indexOf` are linear; no regex over template content. |
</threat_model>
