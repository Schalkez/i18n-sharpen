<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.svelte` source → svelte/compiler | Untrusted-shaped source string parsed by the Svelte compiler (parse-only, no compile/codegen) |
| embedded `<script>` content → parseTypeScriptFile | Delegated parse of sliced script-block text |
| `svelte/package.json` → version gate | Workspace-resolved package metadata read via createRequire (fixed specifier, not a path from file content) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-S1 | Denial of Service | malformed `.svelte` aborts the run | mitigate | `svelteCompiler.parse(...)` is wrapped in try/catch; the error is mapped into `collectedErrors` (D-17) and the function returns normally — one bad file cannot abort a scan. |
| T-03-S2 | Elevation of Privilege | parser executing parsed source | mitigate | `parseSvelteFile` only walks the AST and delegates to `parseTypeScriptFile` (parser-only `ts.createSourceFile`). No `.compile()`, no render/codegen, no `eval` — source is never executed (grep asserts no `eval`/no `.compile(`). |
| T-03-S3 | Tampering | dynamic load of arbitrary module | mitigate | Compiler loaded only via `loadWorkspaceDep('svelte/compiler', cwd)` and version read via `require('svelte/package.json')` — fixed specifiers resolved from the workspace, never paths derived from file content. |
| T-03-S4 | Denial of Service | ReDoS / quadratic blowup | accept | No regex over template content (AST walk only); `source.slice`/`indexOf` are linear. Pathological-size inputs are out of scope (RESEARCH §Security Domain: WASM/large-file exhaustion deferred). |
</threat_model>
